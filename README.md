# 🌌 比喩生成システム (Metaphor Generator)

ユーザーが日常で抱く言語化できない「モヤモヤ・違和感」や「日々の特別なエピソード（喜びや感動）」を抽出し、文学的なキレ味を持つ1行の前衛的な比喩表現へと昇華させるWebアプリケーションです。
単に結果を返すだけでなく、AIが友達のようにフランクなトーンで生成のウラ話を語る解説機能を備え、さらに世界中のユーザーが匿名でリアルタイムに言葉を共有・管理し合えるソーシャルなタイムライン機能を搭載しています。

## 🛠️ システムアーキテクチャ & 技術スタック

* **Frontend / GUI:** Streamlit (Custom CSSによる文学的・モダンなUIデザイン)
* **Server / AI Core:** Gemini 2.5 Flash API (Structured JSON Output / Few-Shot Learning)
* **Cloud Infrastructure:** AWS S3 (boto3) ＆ **Supabase (PostgreSQL / リアルタイムデータストア)**
* **Security:** python-dotenv (環境変数の秘匿化・多層入力バリデーション)

## 📈 アップデート履歴と進化のプロセス

### 🔹 第1弾：コアロジックの構築
ターミナル環境（CUI）にて、Gemini APIとAWS S3を連携させた基本的な比喩生成スクリプトを構築。

### 🔹 第2弾：堅牢性の強化とインフラの永続化
Web公開を見据えたセキュリティガードと、AWS SAAの知識を活かしたデータ設計を導入。
* **多層バリデーション:** `set()` による文字多様性チェック（キー連打等の荒らし対策）、外部ファイル（`ng_words.txt`）と連携したNGワードフィルターの構築（`.gitignore`による秘匿管理）。
* **S3の永続化:** タイムスタンプによる動的ファイル名生成を実装し、クラウド上での上書き問題を解消。

### 🔹 第3弾：GUI化と生成プロセスの可視化
ユーザー体験（UX）とシステム構造を大幅にブラッシュアップ。
* **文学的な世界観のUI/UX:** 明朝体フォント（Noto Serif JP）や白と水色を基調とした、清潔感と上品さのあるお洒落なデザインをカスタムCSSで独自設計。
* **AIの構造化出力（Structured JSON）:** APIの出力を `response_mime_type="application/json"` で完全制御し、システム側での確実なパースを実現。

### 🔹 第4弾：全エピソード対応拡張 ＆ Supabaseデータベースによるタイムライン（最新）
単なる「悩み相談」から「日常のあらゆるエピソード」の比喩化へとコンセプトを拡張し、インフラをさらにモダンに大改造。
* **全エピソード対応へのプロンプト拡張:** 嬉しいこと・感動したことには「色彩豊かで美しい比喩」を、ネガティブなモヤモヤには「鋭く冷徹な比喩」を臨機応変に紡ぎ分ける柔軟なAIロジックを設計。
* **インメモリから外部データベース（Supabase）への移行:** これまでのサーバー内インメモリ（`@st.cache_resource`）によるタイムライン管理の限界（個別削除が不可能、サーバー再起動で全消えする問題）を解決するため、クラウド型PostgreSQL（Supabase）を導入。
* **固有ID（BIGSERIAL）を用いたピンポイント削除機能:** 各投稿に固有の背番号（ID）を自動で割り振ることで、世界タイムライン上から**「特定の投稿だけを狙い撃ちで削除（CRUDのD）」**できる実務レベルの動的制御を実装。

## 💡 技術選定の理由

* **Supabase (PostgreSQL):** テキストデータの永続化と、超高速なリアルタイムタイムライン通信を両立させるため。また、固有IDの管理によってデータごとの柔軟な削除操作を可能にするため。
* **Gemini 2.5 Flash (JSON Mode):** 詩的な感性と冷徹な構造分析という相反する出力を高い解像度で両立させ、かつシステム側で確実にパース可能な構造化データ（JSON）を高速に取得するため。
* **AWS S3:** 拡張性が高く堅牢なオブジェクトストレージインフラに、監査ログとして「元の入力」「生成された比喩」「解説」の全生データをboto3経由でセキュアに永続化（アーカイブ）するため。
