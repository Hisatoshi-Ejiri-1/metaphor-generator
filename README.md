# 🌌 比喩表現生成システム

ユーザーが日常で抱く言語化できない「モヤモヤ」や「違和感」を抽出・構造分析し、極限まで削ぎ落とされた1行の前衛的な比喩表現へと昇華させるシステムです。
裏側では、文学的なアプローチを論理的に制御するプロンプトデザインと、AWS（S3）クラウドインフラへの保存連動を実装しています。

## 🛠️ システムアーキテクチャ & 技術スタック
- **Language:** Python 3.x
- **AI Core:** Gemini 2.5 Flash API (Structured Prompting / Few-Shot Learning)
- **Cloud Infrastructure:** AWS S3 (boto3)
- **Security:** python-dotenv (環境変数の秘匿化)

---

## 🚀 今回のアップデート (第2弾)
システムの安定運用とプロダクション環境への移行を見据え、大幅なロジックの強化とセキュリティ防御を実装しました。

### 1. 核心を突くプロンプトへの刷新
「構造の抽出」「意味の距離の最大化」「五感のハック」の3軸でシステムプロンプトを再設計。AI特有の「説明的な比喩」や「ありきたりな表現」を徹底排除し、キレ味のある前衛詩のような出力を強制するFew-Shotを設定。

### 2. 多層的な入力バリデーション（荒らし・悪質入力対策）
Webアプリケーションとしての一般公開を想定し、手前での不正リクエストのブロック処理を構築。
- **文字数制限:** 100文字上限、5文字未満の短すぎる入力をガード。
- **文字多様性チェック:** `set()` ロジックを用い、「ああああ」などの無意味なキー連打を自動検知してAPIの浪費を防御。
- **外部ファイル連携型NGワードフィルター:** 暴言や不適切ワードを検知するブラックリスト機能を実装。面接官や第三者への配慮および保守性を考慮し、NGワードのソースは外部ファイル（`ng_words.txt`）へ分離させ、`.gitignore` で厳重に秘匿管理。

### 3. AWS S3 履歴管理の永続化 (SAA知識の実践)
保存時のファイル名を従来の固定名から、実行日時に応じた動的ファイル名（例: `20260518_031500.txt`）へ自動生成するよう変更。これによりクラウド上での上書き問題を解消し、データの永続的な蓄積とログ管理を可能にしました。

## 🔮 今後の改善予定 (Next Steps)
就活でのポートフォリオとしての価値、およびWebアプリケーションとしての完成度をさらに高めるため、以下の機能を順次実装予定です。

1. **トレーサビリティの確保（データのセット保存）**
   - S3へ保存する際、「錬成された比喩」だけでなく、トリガーとなった「ユーザーの元のモヤモヤ」もセットで構造化してテキスト保存し、データの後追い（ログ解析）を容易にする。
2. **耐障害性の向上（エラーハンドリングの疎結合化）**
   - Gemini APIの生成処理と、AWS S3へのアップロード処理のエラーハンドリングを分離。万が一インフラ側が瞬断しても、生成されたコンテンツをユーザー画面に返し切る堅牢な設計（可用性の向上）への変更。
3. **Webアプリケーション化 (UI/UXの刷新)**
   - CUI（ターミナル環境）から、Pythonのライブラリ `Streamlit` を用いたリッチなWebサイト（GUI）への移行。入力フォーム、錬成ボタン、おしゃれなテキストエリアの実装。
